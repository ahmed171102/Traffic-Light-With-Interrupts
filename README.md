# 🚦 Traffic Light Controller with Emergency Stop Interrupt

## 📝 Project Overview
This project is a microcontroller-based traffic light system written in C using the **mikroC PRO for PIC** IDE. It controls a standard three-state traffic light sequence (Green ➔ Yellow ➔ Red) and outputs the current state to a 16x2 LCD screen. 

The core feature of this system is a hardware-driven **Emergency Stop** triggered by an external interrupt. To ensure the system is perfectly responsive, it utilizes a custom "Smart Delay" function (non-blocking delay). If the emergency button is pressed at any point during the cycle, the microcontroller immediately breaks out of its delay, halts the sequence, shuts off all LEDs, and displays a "Traffic OFF" message on the LCD until the system is physically reset.

## ⚙️ Hardware Used
* **Microcontroller:** Microchip PIC16F877A (Clock set to 8MHz)
* **Display:** Standard 16x2 Character LCD (LM016L in Proteus)
* **Lights:** 3x LEDs (Green, Yellow, Red)
* **Input:** 1x Push Button (Active High)
* **Passive Components:** 3x 220Ω Resistors (for LEDs), 1x 10kΩ Resistor (Pull-down for the button)

## 🔌 Pinout & Wiring Guide

### The Interrupt Button (PORT B)
* **RB0 (Pin 33):** Connected to the Push Button (tied to +5V). A 10kΩ pull-down resistor connects RB0 to Ground to prevent floating pin issues.

### The Traffic Lights (PORT D)
* **RD0:** Green LED (via 220Ω resistor to Ground)
* **RD1:** Red LED (via 220Ω resistor to Ground)
* **RD2:** Yellow LED (via 220Ω resistor to Ground)

### The LCD Screen (PORT B)
* **RS:** RB4
* **EN:** RB5
* **D4:** RB1
* **D5:** RB2
* **D6:** RB3
* **D7:** RB6

## 💻 Software Features
* **External Interrupt (`INTF_bit`):** Configured to trigger on a rising edge (`INTEDG = 1`). This allows the microcontroller to instantly register the button press regardless of where it is in the main loop.
* **Smart Polling Delay:** Instead of using standard blocking delays (like `Delay_ms(5000)`), the code breaks delays down into 1-millisecond intervals. It checks the `volatile` interrupt flag during every loop iteration, guaranteeing zero lag between pressing the button and the system shutting down.

## 🚀 How to Simulate
1. Open the included Proteus simulation file (`.pdsprj` or `.DSN`).
2. Double-click the **PIC16F877A** microcontroller chip on the schematic.
3. Under the **Program File** field, browse for and select the compiled `.hex` file generated by mikroC.
4. Set the **Processor Clock Frequency** to exactly `8MHz`.
5. Click **OK**, then press the **Play (►)** button in the bottom left corner of Proteus to start the simulation. 
6. Press the button attached to RB0 to trigger the Emergency Stop!
